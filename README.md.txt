# Manual Testing Project – Flight Booking Application

##  Project Overview
This project demonstrates a complete manual testing workflow for a Flight Booking web application using Jira and Zephyr. The purpose is to showcase a structured, real-world QA process with detailed documentation

## Tools & Technologies
- **Jira** for test management and defect tracking  
- **Zephyr** plugin for test case management  
- **Excel / CSV** for exporting test cases  
- **Google docs** for reporting Bugs

##  Project Modules
- Login & Authentication  
- Search Flights  
- Booking Flow  
- Payment  
- Confirmation Page

##  Deliverables
- Test Cases (Functional & Boundary Tests)
- Bug Reports (Critical & UI Bugs)
- Test Execution Summary
- Screenshots of test evidence

## Highlights

-Designed and executed test cases following INVEST, Boundary Value Analysis, and Equivalence Partitioning.
-Linked requirements to test cases using a Traceability Matrix.
-Logged and tracked defects using Jira workflow.
-Prepared Test Execution Summary reports with Zephyr metrics.
